The Slimecoin protocol relies on narrow authority boundaries and explicit accounting state. The short rule is: games calculate scores, matchmaking compares players, and the vault moves balances.

## Authority model

| Authority | Used for | Validated by |
| --- | --- | --- |
| Player wallet | Owns profile and payout accounts | Profile PDA seeds and `authority` field |
| Session authority | Signs gameplay and claims on behalf of the profile | `Profile.session_authority` |
| Game signer PDA | Lets `slimecoin-games` reserve/refund game buy-ins | Vault checks the PDA and game session owner |
| Matchmaking authority PDA | Lets `supersize-matchmaking` refund expired entries and settle matches | Vault checks PDA derived from queue |
| Supersize authority | Administrative setup, accounting, seeding rewards, and emergency flows | Hard-coded authority pubkey |

## Settlement invariants

The vault and matchmaking programs enforce the following:

- player A and player B cannot be the same payout account
- player payout accounts must be owned by the vault
- game sessions must be owned by `slimecoin-games`
- queues must be owned by `supersize-matchmaking`
- the queue validator must match the rollup ledger validator
- regions must match between profile, payouts, and validator
- under-locked matches are voided and released amounts are returned claimable
- draw outcomes refund both buy-ins

## Fee accounting

For a paid USD match:

1. Both players lock the buy-in.
2. The pot is `2 * buy_in`.
3. The rollup platform fee is computed from `platform_fee_bps`.
4. Winner receives `pot - fee`.
5. The fee is logged to the rollup ledger.
6. Global accounting aggregates rollup ledgers and computes the weekly leagues pool.

For the current 10% fee model:

| Pot share | Destination | Why it exists |
| --- | --- | --- |
| 90% | Winner payout | Main competitive prize |
| 5% | SLIME collateral | Supports the free-to-play SLIME supply |
| 3% | Weekly leagues pool | Rewards leaderboard performance |
| 2% | Protocol earnings | Funds operations and protocol growth |

## Rollup reconciliation

`run_accounting` reads all three rollup ledgers and updates the global ledger:

- total SLIME supply
- total SLIME USDC collateral
- total USD fees
- weekly fees
- Slimecoin treasury holdings
- global halving level
- weekly leagues pool

The weekly leagues pool is:

```text
base pool = 60% of weekly_fees
weekly_fees = 50% of collected USD fees
base pool = 30% of collected USD fees
new pool = base pool + unclaimed previous pool + optional bonus
```

## Withdrawal safety

Players withdraw USDC from profile balances through the vault token account. Sponsored accounts pay a one-time withdrawal fee that is recorded in `GlobalLedger.withdrawal_fees`; after the sponsored account pays that fee, the sponsored flag can be cleared.

The protocol authority can withdraw only the bounded protocol share of fees and explicitly claimed withdrawal fees.

## What is intentionally off-chain?

Some presentation and ranking surfaces are off-chain:

- rendered game UI
- API leaderboard indexing
- profile display names and avatars
- social or campaign surfaces

These do not replace on-chain settlement. The authoritative balances, buy-in locks, reward claims, and settlement outcomes are stored in protocol accounts.
