## Shared controls

All games follow the same high-level lifecycle:

1. Join and reserve the buy-in.
2. Initialize or resume a session PDA.
3. Tick gameplay state with compact input bits.
4. Mark the game over by rules or timeout.
5. Submit or resolve the score.
6. Reset the session for the next run.

Games differ in rules, scoring, seeds, and timing, but settlement is deliberately shared.

## Asteroids Blitz

On-chain name: `cannon_game`

Asteroids Blitz is a 3-minute arcade shooter. The player controls a cannon, fires bullets, destroys asteroids, and uses boosters. The program tracks asteroids, bullets, boosters, simulation time, and score.

Key constants:

| Field | Value |
| --- | ---: |
| Max session time | 180,000 ms |
| Tick interval | 33 ms |
| World size | 800 x 600 desktop, 450 x 800 mobile |
| Max bullets | 30 |
| Max asteroids | 8 desktop, 6 mobile |

## Slime Solitaire

On-chain name: `solitaire`

Slime Solitaire is on-chain Klondike. The session stores tableau, waste, stock, foundations, hidden-card state, score, seed, opponent fields, and run metadata.

Scoring includes:

| Action | Score |
| --- | ---: |
| Move to tableau | 20 |
| Move to foundation | 100 |
| Recycle penalty | -20 |
| Move back from foundation | -100 |
| New deal cost | 1,000 |

V3 Solitaire uses shared-seed matchmaking with a 5-minute session window.

## Blob Battle

On-chain name: `blob_battle`

Blob Battle is a real-time arena game. The player grows by collecting food and competing against bots. The program stores a large session account plus a heatmap account for efficient spatial logic.

Key constants:

| Field | Value |
| --- | ---: |
| Map size | 3,840 |
| Sectors | 9 |
| Food per sector | 128 max |
| Bots | 4 |
| Session window | 2 minutes |

Blob Battle uses score-after-play submission and game-specific reset/refund paths.

## Bubble Shooter

Bubble Shooter tracks a 9-column by 13-row bubble board. Players aim shots, bank off walls, pop groups, drop disconnected bubbles, and clear waves.
